sparkrun is designed to be scriptable. This page covers flags, output formats, and patterns for using sparkrun in non-interactive contexts.

## JSON output

Several commands support `--json` for machine-readable output:

```bash
# Job status as JSON
sparkrun status --json

# Cluster health check with JSON output
sparkrun cluster check-job --json

# Cluster monitor (continuous JSON stream)
sparkrun cluster monitor --json
```

## Health checks

Use `sparkrun cluster check-job` to verify a running workload is healthy:

```bash
# Basic check — exits 0 if healthy, non-zero otherwise
sparkrun cluster check-job my-recipe

# JSON output for parsing
sparkrun cluster check-job my-recipe --json

# Include HTTP health check against the inference endpoint
sparkrun cluster check-job my-recipe --check-health
```

## Idempotent launches

The `--ensure` flag makes `sparkrun run` idempotent — if the workload is already running with the same configuration, it's a no-op:

```bash
sparkrun run qwen3-1.7b-vllm --ensure
```

This is useful in scripts and cron jobs where you want to guarantee a workload is running without duplicating it.

## Non-interactive flags

| Flag | Effect |
|---|---|
| `--no-follow` | Launch and exit immediately without following logs |
| `--no-rm` | Keep containers after the workload stops (useful for log inspection) |
| `--ensure` | Idempotent launch — skip if already running |
| `--dry-run` / `-n` | Show what would be done without executing |

```bash
# Launch in background without following logs
sparkrun run qwen3-1.7b-vllm --no-follow

# Keep containers after stop for debugging
sparkrun run qwen3-1.7b-vllm --no-rm
```

## systemd services

Generate systemd unit files for persistent inference services:

```bash
# Export a systemd service file
sparkrun export systemd my-recipe

# Save to a file
sparkrun export systemd my-recipe --save /etc/systemd/system/sparkrun-my-recipe.service
```

The generated unit file uses `sparkrun run --ensure --no-follow` for the start command and `sparkrun stop` for the stop command.

## Exit codes

| Code | Meaning |
|---|---|
| `0` | Success |
| `1` | General error |
| `2` | Invalid arguments or missing recipe |

## Environment variables

| Variable | Purpose |
|---|---|
| `HF_HOME` | HuggingFace cache root directory |
| `HF_HUB_CACHE` | HuggingFace model cache directory (overrides `HF_HOME/hub`) |
| `HF_TOKEN` | HuggingFace authentication token (propagated into containers) |
| `SPARKRUN_SSH_USER` | Default SSH user for all connections |
| `SPARKRUN_LOG_LEVEL` | Logging verbosity (`DEBUG`, `INFO`, `WARNING`, `ERROR`) |

## Scripting patterns

### Launch and wait for health

```bash
#!/bin/bash
sparkrun run my-recipe --no-follow
sleep 10
sparkrun cluster check-job my-recipe --check-health
```

### Ensure a workload is running

```bash
#!/bin/bash
# Idempotent — safe to run repeatedly
sparkrun run my-recipe --ensure --no-follow
```
