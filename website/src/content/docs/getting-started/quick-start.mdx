## Single-node usage

No cluster setup needed — just point sparkrun at a host and run:

```bash
# Run on a single DGX Spark (defaults to localhost if no host specified)
sparkrun run qwen3-1.7b-vllm

# Run on a specific remote host
sparkrun run qwen3-1.7b-vllm --hosts 10.24.11.13

# GGUF quantized models via llama.cpp
sparkrun run qwen3-1.7b-llama-cpp
```

sparkrun launches jobs in the background (detached containers) and follows logs. **Ctrl+C detaches from logs — it never kills your inference job.** Your model keeps serving.

## Multi-node cluster setup

Multi-node inference (tensor parallelism across Sparks) requires a one-time setup. The **setup wizard** handles everything interactively:

```bash
sparkrun setup
```

The wizard walks through each step:

1. **Cluster creation** — saves your DGX Spark hosts by name so you don't repeat host lists
2. **SSH mesh** — establishes passwordless SSH between all hosts and your control machine
3. **CX7 networking** — configures high-speed ConnectX-7 interfaces with static IPs and jumbo frames
4. **Sudoers entries** — scoped rules for cache permissions and page cache clearing
5. **earlyoom** — OOM protection that keeps nodes responsive under memory pressure

Each step is optional — skip any and come back later. When running on a DGX Spark, the wizard auto-detects CX7 peers on your network.

:::tip
Running `sparkrun setup` with no subcommand auto-launches the wizard when no default cluster is configured. You can also invoke it explicitly with `sparkrun setup wizard`.
:::

<details>
<summary>Manual setup (step-by-step)</summary>

If you prefer to configure each step individually:

1. **Create a cluster**: `sparkrun cluster create mylab --hosts 10.24.11.13,10.24.11.14 --user dgxuser` — see [Cluster Commands](/cli/cluster-commands/)
2. **SSH mesh**: `sparkrun setup ssh` — see [SSH Setup](/getting-started/ssh-setup/)
3. **CX7 networking**: `sparkrun setup cx7` — see [Networking Best Practices](/getting-started/networking/)
4. **earlyoom**: `sparkrun setup earlyoom` — see [Setup Commands](/cli/setup-commands/#install-earlyoom-oom-killer)

Set your cluster as default so you don't need `--cluster` on every command:

```bash
sparkrun cluster set-default mylab
```

</details>

## Run an inference job

```bash
# Single node — minimum nodes/parallelism is set by the recipe
sparkrun run qwen3-1.7b-vllm

# Multi-node tensor parallel using your default cluster
sparkrun run qwen3-1.7b-vllm --tp 2

# Override settings on the fly
sparkrun run qwen3-1.7b-vllm --hosts 192.168.11.14 --port 9000 --gpu-mem 0.8
sparkrun run qwen3-1.7b-vllm --tp 2 -H 192.168.11.13,192.168.11.14 -o max_model_len=8192
```

## Inspect a recipe

```bash
sparkrun show nemotron3-nano-30b-nvfp4-vllm
```

This displays recipe details including a VRAM estimation that tells you whether the configuration fits DGX Spark's 128 GB unified memory before you launch.

## Manage running workloads

```bash
# Re-attach to logs (Ctrl+C is always safe)
sparkrun logs nemotron3-nano-30b-nvfp4-vllm --cluster mylab

# Stop a workload
sparkrun stop nemotron3-nano-30b-nvfp4-vllm --cluster mylab
```

:::note
If you launched with `--tp` (modifying the recipe default), pass the same `--tp` flag to `stop` and `logs` so they resolve the same cluster ID:

```bash
sparkrun run nemotron3-nano-30b-nvfp4-vllm --tp 2
sparkrun stop nemotron3-nano-30b-nvfp4-vllm --tp 2
sparkrun logs nemotron3-nano-30b-nvfp4-vllm --tp 2
```
:::

## Spark Arena recipes

[Spark Arena](https://spark-arena.com) is the community recipe hub for DGX Spark. Browse tested recipes and benchmark results, then run them directly:

```bash
# Run a recipe from Spark Arena using its short link
sparkrun run @spark-arena/<recipe-id>

# Inspect it first with VRAM estimation
sparkrun show @spark-arena/<recipe-id>

# Save a copy locally for customization
sparkrun show @spark-arena/<recipe-id> --save my-recipe.yaml
```

Spark Arena recipes are also included in sparkrun's default registries — `sparkrun list` shows them automatically.

## Custom recipe registries

```bash
# See configured registries
sparkrun registry list

# Add a community or private registry
sparkrun registry add https://github.com/myorg/spark-recipes.git

# Update all registries
sparkrun registry update

# Search across all registries
sparkrun search qwen3
```

See [Registries](/recipes/registries/) for more details.
