## Install

```bash
# Install uv if you don't have it
curl -LsSf https://astral.sh/uv/install.sh | sh

# Install sparkrun and set up your cluster
uvx sparkrun setup
```

The setup wizard walks you through cluster creation, SSH mesh, and CX7 networking interactively. See the [Setup Wizard Walkthrough](/getting-started/setup-wizard/) for a detailed step-by-step guide.

## Run a model (single node)

No cluster setup needed — just point sparkrun at a host and run:

```bash
# Run on localhost (or your default cluster's first host)
sparkrun run qwen3-1.7b-vllm

# Run on a specific remote host
sparkrun run qwen3-1.7b-vllm --hosts 10.24.11.13
```

sparkrun launches containers in the background and follows logs. **Ctrl+C detaches from logs — it never kills your inference job.** Your model keeps serving.

## Run multi-node (tensor parallelism)

With a cluster configured, scale across multiple DGX Sparks:

```bash
sparkrun run qwen3-1.7b-vllm --tp 2
```

Each DGX Spark has one GPU with 128 GB unified memory, so `--tp 2` splits the model across 2 nodes. See [Multi-Node Tensor Parallelism](/tutorials/multi-node/) for a full walkthrough.

## Inspect a recipe

```bash
sparkrun show nemotron3-nano-30b-nvfp4-vllm
```

This displays recipe details including a VRAM estimation that tells you whether the model fits in memory before you launch.

## Status, logs, and stop

```bash
# Check what's running
sparkrun status

# Re-attach to logs (Ctrl+C is always safe)
sparkrun logs nemotron3-nano-30b-nvfp4-vllm

# Stop a workload
sparkrun stop nemotron3-nano-30b-nvfp4-vllm
```

:::note
If you launched with `--tp` (modifying the recipe default), pass the same `--tp` flag to `stop` and `logs` so they resolve the same cluster ID.
:::

## Spark Arena

[Spark Arena](https://spark-arena.com) is the community recipe hub for DGX Spark — browse tested recipes and benchmark results, then run them directly with sparkrun. Spark Arena recipes are included in sparkrun's default registries, so `sparkrun list` shows them automatically.
