---
title: Cluster Commands
description: Create, manage, and configure named clusters.
---

Clusters are named groups of DGX Spark hosts. Save your hosts once and reference them by name in every command.

The first host in a cluster definition is used as the **head node** for multi-node jobs.

## Create a cluster

```bash
sparkrun cluster create <name> --hosts <ip1>,<ip2>,... [options]
```

| Option | Description |
|---|---|
| `--hosts` / `-H` | Comma-separated host list |
| `--hosts-file` | File with hosts (one per line) |
| `-d` / `--description` | Cluster description |
| `--user` / `-u` | SSH username for this cluster |
| `--cache-dir` | HuggingFace cache directory on cluster hosts (useful when control machine and cluster have different paths, e.g. macOS → Linux) |
| `--transfer-mode` | Resource transfer mode: `auto`, `local`, `push`, or `delegated` |
| `--transfer-interface` | Network interface for transfers: `auto`, `cx7`, or `mgmt` |

```bash
sparkrun cluster create mylab \
  --hosts 10.24.11.13,10.24.11.14 \
  --user dgxuser \
  -d "2-node DGX Spark cluster"

# With transfer mode for external control machines
sparkrun cluster create mylab \
  --hosts 10.24.11.13,10.24.11.14 \
  --transfer-mode push
```

See [Resource Transfer Modes](/getting-started/networking/#resource-transfer-modes) for details on transfer mode options.

## Update a cluster

```bash
sparkrun cluster update <name> [options]
```

All options are the same as `create`. Only provided options are updated — omitted fields are left unchanged. Pass `--user ""` or `--cache-dir ""` to explicitly clear a value.

```bash
sparkrun cluster update mylab --hosts 10.24.11.13,10.24.11.14,10.24.11.15
sparkrun cluster update mylab --transfer-mode delegated
sparkrun cluster update mylab --user newuser
```

See [Resource Transfer Modes](/getting-started/networking/#resource-transfer-modes) for details on transfer mode options.

## List clusters

```bash
sparkrun cluster list
```

Shows all saved clusters with their hosts and descriptions. The default cluster is marked with `*`.

## Show cluster details

```bash
sparkrun cluster show <name>
```

Displays name, description, user, cache directory, transfer mode, default status, and all hosts.

## Delete a cluster

```bash
sparkrun cluster delete <name>
sparkrun cluster delete <name> --force    # skip confirmation prompt
```

## Set/unset default cluster

```bash
sparkrun cluster set-default <name>
sparkrun cluster unset-default
sparkrun cluster default        # show current default
```

When a default cluster is set, commands like `sparkrun run` and `sparkrun stop` use it automatically without needing `--cluster` or `--hosts`.

## Check cluster status

```bash
sparkrun cluster status
sparkrun status              # alias
```

Shows running sparkrun containers, pending operations (downloads/distributions in progress), and idle hosts across the cluster. See [Cluster Status](/clusters/status/) for full details.

## Monitor cluster

```bash
sparkrun cluster monitor [options]
```

Live-monitor CPU, RAM, and GPU metrics across cluster hosts. By default launches an interactive TUI; pass `--simple` for plain-text output. Press `q` (TUI) or `Ctrl-C` to stop.

![Cluster monitor TUI](/sparkrun-cluster-monitor.jpg)

| Option | Description |
|---|---|
| `--hosts` / `-H` | Comma-separated host list |
| `--hosts-file` | File with hosts (one per line) |
| `--cluster` | Use a saved cluster by name |
| `--interval` / `-i` | Sampling interval in seconds (default: 2) |
| `--simple` | Use plain-text output instead of TUI |
| `--dry-run` / `-n` | Show what would be done |

```bash
sparkrun cluster monitor --cluster mylab
sparkrun cluster monitor --cluster mylab --interval 5
sparkrun cluster monitor --cluster mylab --simple
```
